# Razgon — AI-помощник для продвижения сайтов

## Идея

Веб-сервис для бизнеса: подключаешь Яндекс Метрику и Webmaster, AI-агент анализирует данные сайта и выдаёт персональный отчёт с рекомендациями. Уточняй детали в чате.

## Клиентский путь

### 1. Промо-страница (`/`)
- Описание сервиса в несколько блоков (что даёт, как работает, преимущества)
- Кнопка **«Начать»**
- По кнопке — редирект на Яндекс OAuth с запросом доступа к Метрике и Webmaster

### 2. Авторизация Яндекса
- OAuth-запрос прав: `metrika:read`, `webmaster:read`
- После подтверждения — возврат на сайт с токеном
- Если пользователь новый — создаётся аккаунт

### 3. Выбор сайта (`/setup`)
- Загружаем список сайтов/счётчиков из Метрики
- Клиент выбирает сайт из списка
- Кнопка **«Создать отчёт»**

### 4. Генерация отчёта
**Сбор данных (параллельно):**
- Метрика: трафик, источники, поведенческие метрики, конверсии, популярные страницы
- Webmaster: ошибки сканирования, поисковые запросы, позиции, индексация

**Анализ:**
- Все данные собираются в структурированный промпт
- Отправляются по REST стороннему AI-агенту
- Агент возвращает текстовый отчёт с рекомендациями

**Отображение (`/project/{id}/report`):**
- Текст отчёта от агента
- Стандартные графики (трафик, источники, запросы) — строятся из собранных данных
- Внизу — поле чата для уточняющих вопросов

### 5. Личный кабинет (`/dashboard`)
- Список проектов (по имени сайта)
- В каждом проекте:
  - Сохранённый отчёт
  - История переписки с агентом
  - Возможность обновить данные

## Стек

- **Backend:** Python (Flask)
- **Frontend:** JS, Bootstrap CSS, Chart.js (графики)
- **AI:** Сторонний REST-агент (формат OpenAI совместимый)
- **Auth:** Яндекс OAuth
- **Data:** Яндекс Метрика API, Яндекс Webmaster API
- **DB:** SQLite (SQLAlchemy)

## DB-модели

```
User
├── id
├── yandex_id
├── name
├── email
├── oauth_token (encrypted)
└── created_at

Project
├── id
├── user_id → User
├── site_name
├── metrika_counter_id
├── webmaster_host_id
└── created_at

Report
├── id
├── project_id → Project
├── raw_data (JSON: данные из Метрики + Webmaster)
├── ai_report_text
├── created_at
└── updated_at

ChatMessage
├── id
├── project_id → Project
├── role (user / agent)
├── text
└── created_at
```
